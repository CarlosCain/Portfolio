Project Overview

The PAWS Storefront is a secure, full‑stack e‑commerce application designed to simulate a real-world online shopping experience for pet products. Customers can register, log in, browse inventory, manage a shopping cart, and complete purchases. Administrators can manage product listings, update inventory, and review order activity.

The application is built using Spring Boot, Thymeleaf, Bootstrap, and MySQL, following a layered architecture that separates presentation, business logic, and data access. Security is enforced through Spring Security, bcrypt password hashing, CSRF protection, and HTTPS/TLS for all authenticated routes.

This portfolio summarizes the system’s architecture, design decisions, implementation details, testing results, and final demonstration for the CST‑452 Senior Capstone Project.

High-level system architecture, deployment view, and relational data model for the PAWS Storefront.

System Architecture

The PAWS Storefront uses a layered architecture with a Presentation layer (Thymeleaf), Business Logic layer (Spring services), and Data layer (JPA repositories and MySQL). This structure supports scalability, maintainability, and secure request handling.

  • Presentation: HTML, CSS, Bootstrap, Thymeleaf
  • Business Logic: UserService, ProductService, CartService, SecurityService
  • Data: JPA/Hibernate repositories backed by MySQL

Entity-Relationship Diagram (ERD)

The database schema models users, products, carts, and orders with clear foreign key relationships. This ensures data integrity and supports secure, consistent e-commerce workflows.

  • Users & Authorities: Authentication and role-based access
  • Products: Inventory with pricing and descriptions
  • Orders & OrderDetails: Purchase records and line items
  • CartItems: Session cart persisted at checkout

How to Run the Project

The PAWS Storefront is a Spring Boot web application that runs locally using Java and MySQL. Follow the steps below to set up the project, configure the database, and launch the application.

Prerequisites
  • Java 17 or higher installed
  • MySQL Server installed (local instance)
  • Maven installed (optional if using IntelliJ/VS Code Spring Boot tools)
  • Git installed to clone the repository
1. Clone the Repository
git clone https://github.com/CarlosCain/PAWS-Storefront.git
cd PAWS-Storefront
2. Configure the Database

Create a MySQL database named paws and update your credentials in src/main/resources/application.properties:

spring.datasource.url=jdbc:mysql://localhost:3306/paws
spring.datasource.username=YOUR_USERNAME
spring.datasource.password=YOUR_PASSWORD
spring.jpa.hibernate.ddl-auto=update
3. Run the Application

You can run the project using your IDE or from the command line:

mvn spring-boot:run
4. Access the Application

Once running, open your browser and navigate to:

https://localhost:8443
Default Login Credentials
  • User: user / password123
  • Admin: admin / admin123

These accounts allow you to explore both customer and administrative workflows, including browsing products, managing the cart, checking out, and performing admin CRUD operations.
